安裝Vue Cli環境

前言:
上完課程後覺得環境建立步驟有點多，怕久了忘記來記錄一下 ，
載入前請先安裝node.js與npm並確保版本。

開始安裝

第一步安裝Vue Cli:

在終端機內輸入npm install -g @vue/cli
安裝完成後可以輸入 vue-V 查看是否跳出目前cli版本 ，有出現表示安裝成功。

第二步指定到專案路徑:

在終端機內指定使用cd "你的專案路徑名稱" 指定到路徑後就可以開始建立專案囉

第三步創建一個項目:

指定到路徑後使用vue create new-project 來建立專案new-project是自訂資料夾名稱，可替換。

第四步設定環境預設選項

在這些選項中，我們將會選擇Manually select features 這個選項是用手動來建立我們的環境。

接著進去下個選項這裡有幾樣需要點選 指列出目前需要的，沒用到先不列出。

  • Babel

    會幫你把ES6，7等等以上版本編譯為大部 分瀏覽器可運行的JS

  • Router

    這個要點選，但還沒學到，之後補上

  • Vuex

    資料狀態管理工具，以後會用到，現在先不選。

  • CSS Pre-processors

    這個要點選，但還沒學到，之後補上

  • Linter / Formatter

    這個可以檢視你的城市碼品質工具，也是開啟。

接下來跳會出下一頁，以下選項都是依照之前我們所選擇的那些項目來繼續設定。

  • Choose a version of Vue.js 3.x

    這個是選擇Vue版本，這個範例中選版本3

  • Use history mode for router?

    選擇Vue router形式，這裡先選n，使用預設形式。

  • Pick a CSS pre-processor

    選擇CSS的預處理器，由於目前版本更新後node-sass選項已經是預設不會出現，所以選dart-sass

  • Pick a linter / formatter...

    ESLint選擇撰寫風格，有兩種Airbnb和Standard， 剛開始可以先使用Standard熟練後可以轉Airbnb

    Airbnb : 高手模式，各項JS細節要求特別高。

    Standard : 普通模式，JS掌握度不高可以先用。 我就爛

  • Pick additional lint features

    選擇Lint使用哪種方式檢視，請選擇Lint on save 存檔時檢視

  • where do you prefer placing

    選擇上列設定檔使用是獨立檔案in dedicated config files 或存在package.json 範例中選擇in dedicated config files

  • Save this as a preset...

    最後會詢問你是否需要把設定檔給儲存， 此範例是先不儲存NO

    • 安裝成功後會出現

    cd new-project

    npm run serve

    這時代表你成功了，這時使用VsCode來開啟專案， 在VsCode終端機內輸入npm run serve來運行環境， 到此就全部完成了。

    PS : 如果在過程中不幸選錯了，可以終端機內使用Ctrl + C 來進行中斷後重新操作。
